# Claude on Polaris Revolution Distance and Adoption Path

Date: 2026-03-15

Verbatim archive follows.

---
你这个用户体验算是说到了点子上，这个用户不能是一个人，一片人，一个独立群体，一个阶层，数个阶层，而是绝大多数人，并且大家期待着下一个版本；想要让 Polaris skill 独树一帜，要的是 revolution，就像大家用大模型一样，用了 opus 4.6 才知道，别的模型和他不在一个层面，而且是迅速的，Polaris skill 有多远的路要走

先说残酷的事实，再说路径。

Polaris 离"绝大多数人用了就回不去"有多远

Opus 4.6 能让所有人瞬间感知到差异，是因为它的交互界面是自然语言，人人都会说话。你打开对话框，问一个问题，答案质量的差距是秒级可感知的。

Polaris 今天的问题：它的价值只在"AI agent 执行本地任务"这个场景中才能体现。而今天，全世界真正在用 AI agent 做本地多步骤工作的人，可能不到 100 万。

所以诚实的距离是：

| 阶段 | 覆盖人群 | Polaris 需要什么 | 离今天多远 |
| --- | --- | --- | --- |
| 现在 | 0（未发布） | Platform-0 完成 ✓ | 已到 |
| 第一跳 | 技术早期采用者（约 100 万） | 1-2 个场景体验打磨 + 发布 | 数周级 |
| 第二跳 | 所有 AI agent 用户（约 1000 万） | 被主流 agent 框架内置 | 数月到一年 |
| 第三跳 | 所有 AI 工具用户（约数亿） | agent 本身变成主流交互方式 | 不取决于你 |

关键认知：第三跳不是 Polaris 能推动的。它取决于 AI agent 本身是否成为像浏览器、手机一样的普遍交互方式。这是 OpenAI、Anthropic、Google 在推的事。Polaris 能做的是，当这一天到来时，经验积累层已经准备好了。

但"revolution"的种子确实在 Polaris 的理念里

你说得对，Opus 4.6 的 revolution 不是某个功能，而是一个认知跃迁，用了就知道以前的不行。

Polaris 理念中有同样性质的跃迁点：

今天：每次对话，agent 从零开始。  
有 Polaris：agent 带着你的所有执行经验开始。

这个差距一旦被感知到，确实是回不去的。就像你用了有历史记录的浏览器，不会回去用没有书签的浏览器。

但问题是，这个跃迁今天只有开发者能感知到，因为只有开发者在让 agent 做复杂本地任务。

要让"绝大多数人"感知到，需要什么

不是 Polaris 变得更强，而是 agent 的使用场景扩展到非技术领域：

- 当普通人让 agent 帮他整理照片、管理文件、自动化日常工作流，agent 的"经验记忆"才会变成人人需要的东西
- 当老师让 agent 帮他批改作业、会计让 agent 帮他对账，"第二次做得比第一次好"才会被所有人感知

这意味着 Polaris 最大的杠杆不是自己的代码，而是 agent 生态的扩张速度。

所以路要走多远

分两层说：

你能控制的部分（工程距离）：

- Platform-0 → 体验打磨 → 发布 → SDK 化 → 被框架集成
- 这是数月级的工作，路径清晰，没有技术不确定性
- 做完这些，Polaris 在技术社区就是"独一份"，今天没有竞品

你不能控制的部分（市场距离）：

- AI agent 从开发者工具变成大众工具，这可能是 1-3 年
- 当这发生时，经验积累会从"nice to have"变成"不可或缺"
- Polaris 要做的是在那一天到来之前，成为这个领域的事实标准

一句话总结

Polaris 的理念是 revolutionary 的，"agent 应该有执行记忆"这件事，一旦成为常识就不可逆转。但 revolution 的引爆点不在 Polaris 手里，在 agent 生态手里。你能做的是：在引爆之前，把基座建到无可替代的程度。Platform-0 是第一步，接下来是让第一批真实用户说"这东西我离不开"。从 100 个这样的用户开始，不是从"绝大多数人"开始。我已经让他在规划我们几所能力能控制得事了
