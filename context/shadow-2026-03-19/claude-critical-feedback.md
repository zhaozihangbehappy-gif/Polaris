# Claude 对 Shadow 的批判性、建设性意见

Claude 的核心判断是：**Shadow 当前最大的敌人不是技术难度，而是定义膨胀。**

他认为，这轮 brainstorming 的方法论本身是对的：通过不断二选一、强制排序，试图把一个高度发散的愿景往可执行方向压缩。但真实的问题在于，决策者几乎每一步都在抵抗收敛：不愿按人群收口、不愿为 B / D 排主次、不愿砍掉任务协同、不愿减少首发模式，最后甚至明确说“不砍”。Claude 认为，这暴露出一个本质问题：当前脑中的 Shadow 还不是一个“产品”，而是一个愿景体系。如果这个认知落差不被正视，后续所有 spec 都可能在执行阶段反复被推翻。

Claude 特别指出，当前被同时接受的约束组合在工程和商业上几乎不可共存：

- 头盔和外骨骼双产品同时首发
- 两者都要独立成立
- 头盔还要和普通非智能头盔卷价格
- 外骨骼又要压在 2000–5000 元
- 风险预警与避险要做成平台级能力
- 头盔要承担信息中枢角色
- 甚至还想保留多场景和任务协同

这些约束放在一起，不是“很难”，而是高度冲突。尤其危险的是，关于“为了便宜愿意牺牲什么”这个关键问题，被用“以后再议”绕过去了。Claude 认为，这意味着功能讨论还没有真正受到 BOM 成本、认证、续航和材料现实的约束，仍然停留在空中楼阁阶段。

他还指出了几个被忽略但其实应该更早进入核心讨论的问题：

1. **认证与法规**：头盔如果进入骑行/通勤场景，认证不是后话，而是会直接反过来决定结构、电池方案、重量分布和成本。
2. **续航与热管理**：风险预警、环境感知、导航、状态监测都意味着持续在线，头盔如果既要轻又要便宜，续航会成为硬墙。
3. **佩戴意愿**：用户为什么要多戴一顶头盔？不是“它更智能”就够了，而是必须回答“它到底解决了什么手机和现有头盔解决不了的问题”。
4. **社会接受度**：外骨骼不是普通消费品，它有强烈的心理门槛和场景门槛，这在极客市场里仍然要面对。

Claude 对头盔产品的关键提醒是：必须回答 **“它比手机多了什么”**。如果所谓信息中枢只是把手机上的导航、消息、状态搬到头上，那消费者没有理由为此增加一个设备。头盔必须建立在手机做不到、或者在该位置做不好的价值上，比如免手持危险感知、视野内即时提醒、低干扰安全辅助等。

他的建设性建议非常明确：

- **不要双产品同时深做**，优先把 Shadow Helmet 当第一阶段主项目，Exo 和 Link 暂时只做架构预留。
- **不要首发就做三四种模式**，而是先选一个极窄但天然需要头盔的场景打穿，例如骑行/摩托通勤安全增强。
- **立刻做逆向 BOM 定价练习**，从价格倒推功能边界。
- **把“比手机多了什么”提升为头盔是否成立的首要问题。**
- **接受 Helmet / Exo / Link 的拆法**，但今天只把 Helmet 的首版 spec 做深，其他两个只保留接口级定义。

Claude 的结论可以压成一句：

> Shadow 当前最大的风险，不是你们做不出来，而是你们还没有真正决定先做什么、先不做什么。
