# Claude + Codex 观点汇总

如果把 Claude 和 Codex 的意见压成一个共同结论，那就是：**Shadow 当前最大的风险不是能力不够，而是定义失控。**

Claude 更强调硬件现实：价格、认证、续航、BOM、佩戴意愿、社会接受度，以及“头盔到底比手机多了什么”。它的判断是，双产品同时首发、两边都要独立成立、还要压极低价格并承载高功能密度，这在工程与商业上高度冲突。因此必须接受拆分，并优先把 Helmet 做深，把 Exo 与 Link 暂时留在架构层。

Codex 更强调系统现实：边界、约束、记忆治理、权限、可审计性、显式状态。它提醒，不要把愿景、Agent 能力、记忆系统、工具编排、长期自治混成一个模糊的大词。真正可持续的 Shadow，必须建立在明确边界、可验证计划、硬约束和失败归因机制之上。

两者都反对继续“什么都想要”的推进方式，也都支持把项目拆成 Helmet / Exo / Link 三段逐个做 spec，并且都认为第一阶段最值得深挖的是 **Helmet**。

## 合并后的最重要建议

1. **先收窄，不要再扩定义。**
2. **接受 Helmet / Exo / Link 三段拆法。**
3. **优先做 Helmet 的首版 spec。**
4. **先证明一个最小闭环为什么成立，而不是同时证明整个 Shadow。**
5. **把“比手机多了什么”与“目标价格下能不能造出来”作为最优先问题。**
6. **补上硬约束：何时自主、何时提问、记忆怎么分层、失败怎么归因。**

## 一句话压缩

Shadow 不是不能大，而是必须先接受：**它只能靠一个被严格收窄的起点长出来，而不能靠一次性把全部愿景装进去。**
