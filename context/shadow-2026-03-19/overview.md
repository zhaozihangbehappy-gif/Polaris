# Shadow 一页式摘要

## 项目定义
Shadow 当前被讨论为一个消费级“全身智能穿戴设备”方向，核心结构不是单个产品，而是三段式：

- **Shadow Helmet**：消费级智能头盔
- **Shadow Exo**：轻量外骨骼
- **Shadow Link**：头盔 × 外骨骼的协同层

## 已收束出来的关键约束

1. **身头协同，但必须可拆开单卖**
   - 头盔单卖要成立
   - 外骨骼单卖也要成立
   - 协同层必须带来额外增益，而不是互相依赖

2. **首个用户锚点**
   - 被压到：**极客个人增强**

3. **价值主轴**
   - 最后收束为：**信息增强 + 安全/保护增强**
   - 不主打夸张力量放大

4. **平台级能力**
   - 风险预警与避险不能依赖头盔
   - 必须被定义为平台级能力，而不是头盔私有功能

5. **价格纪律极强**
   - 外骨骼目标：**2000–5000 元**
   - 头盔首发必须非常便宜，未来要和普通非智能头盔竞争
   - 最不能接受的失败：**贵到卖不动**

6. **市场入口策略**
   - 两个都首发，但**主推头盔**，外骨骼弱曝光

7. **头盔角色定义**
   - 不是“显示更多信息”
   - 而是：**个人移动场景下的实时信息中枢**
   - 身体与状态信息是底层常驻层，环境风险、导航、通信等按场景切换到前台

8. **首发形态**
   - 长期野心：通用信息中枢
   - 首发更像：**场景化切换设备**

9. **关键结论**
   - 这不是一个单一 spec 能解决的问题
   - 必须拆成 Helmet / Exo / Link 三个子项目逐个做 spec

## Claude 的主批评

- 最大风险不是技术，而是**定义膨胀**
- 双产品同时首发 + 极低价格 + 高功能密度，几乎不可能同时满足
- 缺失关键硬件现实约束：认证、续航、BOM、佩戴意愿、社会接受度
- 头盔必须回答：**比手机多了什么**
- 建议优先只把 Helmet 做深，且先打穿一个极窄场景

## Codex 的主批评

- 最大风险是**概念混装**与边界失焦
- 不能把愿景、Agent 能力、记忆系统、工具编排、自治能力全混着推进
- 不能默认模型能力是稳定基座
- 缺失硬约束：何时自主、何时提问、哪些记忆可长期保留、外部动作确认门槛、失败重试边界
- 建议先做一个 **Shadow v0 定义文档**，只写目标用户、核心场景、非目标、硬约束、禁止行为

## 当前最重要的一句话

Shadow 不是不能大，而是必须先接受它**不能同时完整**。
