## Promotion Path

Candidate promotion remains available but is no longer the main roadmap. A candidate can become A-tier only after:

1. A real agent run reproduces the authored failure and reaches the fix.
2. `eval/evidence_writer.py` writes clean `verified_live` evidence.
3. The candidate record is promoted into `experience-packs-v4/`.
4. `scripts/pattern_validator_v4.py` reports it under A-tier.

The promotion path exists for selective high-value cases, not for bulk count chasing.
